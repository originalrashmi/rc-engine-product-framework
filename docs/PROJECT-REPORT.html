Overview

Hits and Misses

HIT What Worked

  • Full 8-phase pipeline executed successfully -- Illuminate through Compound, all gates approved
  • 27 tasks built and committed -- auth, dashboard, invoices, clients, time tracking, cross-cutting services, observability
  • PRD → Tasks → Code pipeline -- 4 PRDs generated, master task list created, code generated from task specifications
  • Post-RC gate passed -- security scan and observability checks completed
  • Child PRD generation -- UX child PRD, time tracking child PRD, and invoice creator child PRD were spawned from the master PRD
  • Seed data generated -- realistic sample data for immediate local development

MISS What Didn't Work

  • Pre-RC phase was skipped -- rc_init was designed to detect new projects and route to Pre-RC research first (prc_start), but during the demo it went straight to the RC Method build phase. This means no market research, competitive analysis, or Cynefin complexity classification was performed before building.
  • UX scoring and audit were skipped -- the ux_score tool was never called during the Define phase, which meant ux_audit was never triggered during Validate. UI components were built without design token enforcement or accessibility checks. This is a known issue documented as COE-001.
  • UI components lack brand design system -- because UX tools were bypassed, the generated UI uses default Tailwind styles rather than a project-specific design system with defined colors, typography, and spacing tokens.

Fixes Shipped After Demo (v1.1.0)

The following changes were committed to address gaps found during and after the demo. All changes are in commit c083839 (March 4, 2026, 9:42 PM EST). 27 files changed, 1,719 insertions.

FIX Gateway

  • rc_init unified entry point -- new cross-domain gateway tool that detects project state across all 4 domains and routes to the correct next tool. Defaults to Pre-RC research (prc_start) for new projects. Includes skip_research opt-out parameter.
  • State detection cascade: Post-RC → RC (phase-aware) → Pre-RC complete → Pre-RC in progress → new project

FIX Security & Stability (8 fixes)

  • Windows paths (C:\...) now accepted in MCP tools (previously rejected)
  • Tier enforcement now fail-closed (unknown tiers default to free, not full access)
  • Gated tools without project_path blocked instead of bypassing tier checks
  • prc_stress_test correctly mapped to stressTest feature
  • Single source of truth for tool-tier mapping (tool-requirements.ts)
  • Activity audit log at .rc-engine/audit/activity.jsonl (append-only)
  • Web server writes .rc-engine/tier.json on tier change (bridges web-MCP enforcement)

FIX New Capabilities

  • Edge Case Analysis Module (Pro) -- automated edge case detection across 7 categories (input-boundary, error-state, concurrency, data-integrity, integration, state-transition, performance-edge). 3-layer analysis with auto-generated remediation tasks.
  • npm distribution -- npm install -g rc-engine or npx rc-engine
  • Startup banner + update check -- version display, non-blocking npm registry check
  • Opt-in telemetry -- anonymous usage data (OFF by default), never collects project data or API keys

Web UI: Current Status

The RC Engine includes a web-based Pipeline Dashboard (React + Vite) with 7 pages: Landing, Login, Dashboard, Pipeline, Settings, Wizard, and Value Report. This UI is included in the public repository.

Important: The web UI is available in the public repo but has not been fully tested. It was originally built for the Pro tier and was unintentionally exposed when the repo was made public. The UI is functional for exploration, but Pro-tier features are gated behind payment via Stripe integration (checkout, webhook, subscription management). Without Stripe configuration, paid features will not activate. The tier enforcement system (tier-guard.ts) defaults all users to the free tier unless a valid .rc-engine/tier.json file exists with a paid subscription.

Pricing Model: Hybrid Subscription + Credits (Revised)

Note: The original pricing in tiers.ts ($29/$79/mo) is being revised based on competitive analysis. RC Engine uses BYOK (users pay their own API costs), so the subscription should be lower than competitors who include tokens. The new model below reflects this decision.

Model: Low-cost subscription for access + included pipeline credits. Extra credits available for purchase. Users bring their own LLM API keys (no markup on token costs).

Quick Start

  1. Install via npm: npm install -g rc-engine or run with npx rc-engine
  2. Add to your MCP client configuration (Claude Code, Cursor, etc.)
  3. Start with rc_init -- it will detect your project state and route you to the right tool
  4. For new projects, it defaults to Pre-RC research first (use skip_research: true to go straight to build)
