## Voice vs Tone

**Voice** = the product's permanent personality. It doesn't change across contexts.
**Tone** = how the voice adapts to different situations.

## Voice Definition Process

### Step 1: Define Voice Attributes

Choose 3-5 adjectives that describe the brand personality.

### Step 2: Define Voice Through Contrasts

For each attribute, define what it IS and what it ISN'T:

```
We are CONFIDENT, not arrogant.
We are DIRECT, not blunt.
We are WARM, not sappy.
```

### Step 3: Create a Voice Chart

| Attribute | Do This | Not This |
|-----------|---------|----------|
| Confident | "This saves you 3 hours per week" | "We believe this might help save time" |
| Clear | "Your data syncs every 5 minutes" | "Leveraging real-time data synchronization" |

## Tone by Context

| Context | Tone Shift | Example |
|---------|-----------|---------|
| **Hero/Marketing** | Enthusiastic, bold | "Transform how your team ships" |
| **Feature descriptions** | Confident, specific | "Real-time sync across 12 integrations" |
| **Onboarding** | Warm, encouraging | "Great start! Let's set up your first project" |
| **Error messages** | Calm, helpful | "That email doesn't look right. Try name@company.com" |
| **Success states** | Celebratory but brief | "Done! Your report is ready to share" |
| **Empty states** | Motivating | "No projects yet. Create one to get started" |

---

## Universal DON'Ts

| Rule | Reason |
|------|--------|
| Never say "simple" or "easy" | Dismisses user effort |
| Never say "just" in instructions | Minimizes complexity |
| Never use "please" in button labels | Wastes space |
| Never use exclamation marks in errors | Errors are not exciting |
| Avoid "click here" | Not accessible, not descriptive |
| Avoid "submit" as a button label | Generic; use the specific action |
